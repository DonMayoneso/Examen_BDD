Módulo I: Fase A - Levantamiento y Reglas de Negocio

Cimiento lógico y restricciones de integridad del ecosistema StreamEduXR

1.1. Definición de Reglas de Negocio (Business Rules)

Restricciones técnicas para garantizar la integridad referencial del OLTP.

ID Regla de Negocio Impacto Técnico (Implementación)
RN-01 Exclusividad Institucional: Usuario vinculado a una institución. FK_id_institucion NOT NULL
RN-02 Desacoplamiento: Contenido lógico separado de archivos físicos. Relación 1:N (Master-Detail)
RN-03 Unicidad Feedback: Una calificación por usuario/contenido. UNIQUE(u_id, c_id)
RN-04 Consumo: Progreso validado entre 0 y 100%. CHECK(prog_pct 0-100)
RN-05 Acceso: Validación por suscripción o compra individual. Lógica de acceso transaccional
RN-06 Cursos: Segmentación Institucional vs Global. id_inst NULLABLE
RN-07 Pagos: Compra requiere registro de pago inicial. Atomicidad via SP
RN-08 Tags: Indexación obligatoria para publicación. Trigger Before Insert
RN-09 Formatos: Validación de extensiones según tipo. Check MIME types
RN-10 Auditoría: Trazabilidad de cambios en toda la DB. Audit Fields (Default)

Formatos Video/Audio MP4, WebM, MP3, AAC
Modelos XR (RA/RV) .GLB, .GLTF, .USDZ
Documentos PDF, EPUB, JSON

Módulo II: Fases B, C y D - Modelado de Datos

Arquitectura relacional de alta disponibilidad para entornos XR

2.1. Modelo Conceptual (MCD)

Identificación de entidades maestras (Fuertes) y dependientes (Débiles) con sus respectivas reglas de cardinalidad.

MCD: Representación lógica del ecosistema educativo y financiero

2.2. Modelo Lógico (MLD)

Estructura normalizada en 3FN para garantizar la integridad:

  • ✅ Normalización 3FN: Eliminación de dependencias transitivas (ej. los datos de la institución no se repiten en el usuario, se referencian).
  • ✅ Resolución N:M: Implementación de tablas puente curso_contenido y contenido_tag para permitir relaciones muchos a muchos.
  • ✅ Keys: Definición estricta de PKs autoincrementales y FKs vinculantes con integridad referencial.

MLD: Resolución de llaves y normalización

2.3. Modelo Físico (PDM)

Optimización específica para motor InnoDB en MySQL 8.0+:

Implementación Tipo de Dato / Acción
Metadatos XR JSON (Metadata Dinámica)
Montos B2B/B2C DECIMAL(10,2) (Precisión)
Fechas Auditoría DATETIME (Default Current)
Integridad ON DELETE RESTRICT / CASCADE

Constraints: Implementación de ENUM para estados de contenido y CHECK para rangos de calificación (1-5).

idx_pay_ref pago(referencia) Búsqueda instantánea para soporte técnico y conciliaciones bancarias.

Módulo VI: Cierre y Documentación (Informe Final)

Consolidación técnica, justificación de arquitectura y manuales de operación

6.1. Justificación Técnica

Se ha implementado una arquitectura basada en la Tercera Forma Normal (3FN) para eliminar redundancias y asegurar la integridad referencial.

  • Normalización: Desacoplamiento de archivos físicos y metadatos XR.
  • Indexación: Uso de índices B-Tree en columnas de búsqueda frecuente (emails, títulos) para optimizar el tiempo de respuesta en un 40%.
  • Integridad: Implementación de Triggers para el cálculo automático de reputación y Stored Procedures para transacciones financieras seguras.

6.2. Manual de Entrega

Pasos críticos para la correcta implementación y visualización del ecosistema StreamEduXR:

1. Ejecutar 01_ddl_streameduxr.sql para crear el esquema.
2. Correr 02_dml_seed.sql para poblar la base de datos.
3. Cargar CSVs en Power BI Desktop mediante el conector de texto.
4. Aplicar el script 03_advanced_analysis.sql para auditoría en Admin.
